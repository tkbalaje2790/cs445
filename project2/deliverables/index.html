Project 2: Image Quilting

Part 1: Quilting

Following the process described by Efros and Freeman, image quilting was implemented incrementally; first by taking random tile samples from the texture, then by overlapping tiles that minimize SSD error, and finally by "cutting" the overlap regions along the minumum error boundary. Results from random, simple, and cut quilting are shown below. Generally, random quilting failed to produce smooth edges but was surprisingly decent in some scenarios. Simple quilting stiches together patches from a sample that has a low SSD with the input image. It worked quite well in the textures I tested, but sometimes produced sharp boundaries when it struggled to find a good fit. Cut quilting did a better job of blending the overlapping regions.

Seam Finding

Seam finding using the minimum boundary cut builds upon the overlap method but joins the two patches along the minimum error path, as illustrated below. The third image in each subplot demonstrates the path through the overlap region that minimizes average error across the RGB channels. The seam finding section is passed a "cost" image quantifying the fit of each patch.The process allows for further smoothing of error along overlaps, and produces some good results.

Part 2: Texture Transfer

The cut quilting implementation can be extended to perform texture transfer, allowing for the application of a sample texture on a target image. Efros and Freeman explain that this can be done by adding an additional cost using a correspondance map between the source and target images. I used the luminanace channel to compute the cost image (in addition to the previous overlay cost). An alpha parameter was added to tune the effects of the two cost terms.
